# Day 9: Next.js Setup & JS ES6

## What I Built
I initialized the frontend monorepo using Next.js, configured it with TypeScript and Tailwind CSS, and cleared out the Vercel boilerplate to establish a clean slate.

## What I Learned
* **React Evolution:** React isn't just about the browser anymore. Next.js pushes React back to the server, blending the best of old-school HTML generation with modern interactivity.
* **ES6 Syntax:** Arrow functions (`() => {}`) look much cleaner than the old `function` keyword, and using them for React components makes the codebase feel  modern.

## What Confused Me
The sheer amount of files generated by `create-next-app` (like `tsconfig.json`, `postcss.config.js`,`next.config.ts`) was initially overwhelming compared to a simple Python script.

## How I Solved It
I realized these are just configuration files for the build tools. As a developer, primary focus remains inside the `src/app/` directory. The rest of the files just tell the compiler how to interpret my code.

## My Analogy
Using basic JavaScript/React is like building a car without a blueprint; you might put the steering wheel on backward, and you won't know until you try to drive it (Runtime errors). TypeScript in Next.js is like having a robotic assembly line. The robots (the compiler) check every single bolt and wire before the car even leaves the factory floor.
